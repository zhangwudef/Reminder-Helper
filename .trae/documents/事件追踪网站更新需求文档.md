# 事件追踪网站 - 更新版产品需求文档

## 1. 产品概述

一个简洁的个人事件记录与管理网站，帮助用户追踪日常事件并整合重要信息推送。通过直观的日历视图、数据可视化和信息整合功能，让用户更好地管理时间和重要事项。新增登录功能（通过微信二维码登录），以支持用户数据持久化存储和跨设备同步；新增首页作为入口，提供功能介绍和快捷导航。
目标用户：需要个人时间管理和事件追踪的个人用户。支持微信登录后数据云端存储，无需登录也可本地使用基本功能，但登录后可解锁完整功能（如信息整合推送）。

## 2. 核心功能架构

### 2.1 用户角色

未登录用户：使用本地存储，支持基本事件创建和查看，但信息整合推送功能不可
用。
登录用户：通过微信二维码登录，使用云端存储，支持所有功能，包括信息整合推送和数据同步。

### 2.2 功能模块

事件追踪网站包含以下主要页面：

1.首页：网站入口，介绍功能、展示预生成数据、快捷键跳转到登录页面。
2.登录页面：微信二维码登录，支持扫码授权。
3.事件展示页面：主面板，展示事件列表，支持快捷键跳转到创建事件页面、日历视图和数据整合页面。
4.事件创建页面：创建和管理事件，支持普通事件和信息整合推送事件（特别事件）。
5.日历视图页面：展示所有事件的日期分布，清晰标记当天/某天待办事项。
6.数据整合页面：可视化展示事件统计，针对信息整合推送事件记录爬取信息，支持筛选。

## 2.4 信息整合推送事件（特别事件）细节

&#x20;

* 使用文心一言API：输入事件描述后，API进行上下文理解（例如，NLP分析关键词），爬取相关信息（例如，新闻、数据统计），整合成摘要推送。
* 推送：根据用户选择的推送方式和频率，发送整合信息（例如，邮件包含摘要链接）。
* 存储：整合信息记录在数据整合页面，便于用户后续筛选和查看。

## 3. 核心流程

### 用户操作流程

1. **首页进入**：访问首页 → 查看功能介绍和预生成数据 → 使用快捷键/按钮跳转到登录页面。
2. **登录流程**：进入登录页面 → 扫描微信二维码 → 授权登录 → 跳转到事件展示页面（数据同步）。
3. **事件管理流程**：从事件展示页面 → 快捷键跳转到创建事件页面 → 填写信息（普通或特别事件） → 对于特别事件，调用文心一言API整合信息 → 设置提醒和推送 → 保存。
4. **查看日历流程**：从事件展示页面 → 快捷键跳转到日历页面 → 查看月历和每天待办 → 筛选事件。
5. **数据分析流程**：从事件展示页面 → 快捷键跳转到数据整合页面 → 查看图表和整合信息 → 使用筛选工具。

## 4. 用户界面设计

### 4.1 设计风格

* **主色调**：纯黑色（#000000）和纯白色（#FFFFFF）。
* **辅助色**：浅灰色（#F5F5F5）用于背景和分隔。
* **按钮样式**：简洁矩形设计，悬停时有轻微阴影效果。
* **字体**：系统默认无衬线字体，标题18px，正文14px。
* **布局风格**：卡片式布局，内容分区清晰。
* **图标风格**：线性图标，保持简洁一致性。
* **新增**：首页使用大标题和图标网格布局；登录页面居中二维码显示。

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|---------|
| 首页 | 介绍区 | 大标题、图标列表、预生成数据卡片。 |
| 首页 | 导航区 | 快捷键提示（如"L: 登录"），登录按钮。 |
| 登录页面 | 二维码区 | 居中大二维码，加载动画；登录成功提示。 |
| 事件展示页面 | 列表区 | 表格或卡片显示事件，快捷键提示（如"C: 创建"）。 |
| 事件创建页面 | 表单区 | 白色卡片背景，黑色边框输入框，15px圆角；API调用按钮显示加载中。 |
| 事件创建页面 | 进度条 | 细线型进度条，黑色填充，支持拖拽调整。 |
| 事件创建页面 | 图片上传区 | 虚线边框上传区域，拖拽时边框变黑，预览图自适应大小。 |
| 日历视图页面 | 月历网格 | 7x6网格布局，当天高亮显示，事件用灰度圆点标记；点击弹出待办列表。 |

| 数据整合页面 | 图表区域 | 白色背景，黑色图表线条，网格线用浅灰色。 |

### 4.3 响应式设计

* **桌面优先**：主要面向桌面端用户，优化大屏幕体验。
* **移动端适配**：支持平板和手机浏览，首页和登录页面自适应布局。
* **触控优化**：按钮和交互元素适合触控操作。

### 4.4 动效设计

* **页面切换**：淡入淡出效果，持续时间300ms。
* **按钮交互**：悬停时轻微放大105%，阴影加深。
* **表单验证**：错误提示从下方滑入，持续2秒。
* **数据加载**：骨架屏过渡，避免突兀的内容出现；API调用时显示进度圈。
* **快捷键**：按键时轻微反馈动画（如按钮闪烁）。

## 5. 预生成数据

系统初始化时包含两条示例数据：

1. **普通事件**："团队周会"，重要程度中等，进度50%，设置每周提醒。
2. **信息整合事件**："项目进度汇总"，重要程度高，进度30%，设置每日推送项目相关数据。 这些数据在首页展示，帮助用户理解系统功能，可直接编辑或删除。

## 6. 技术要求

* **登录集成**：使用微信开放平台API生成二维码，支持OAuth授权。
* **API集成**：文心一言API用于特别事件的信息整合（上下文理解和信息爬取）。
* **数据存储**：未登录使用本地存储（LocalStorage）；登录后使用云端数据库（例如，Firebase或自定义后端）。
* **推送服务**：集成第三方服务（如OneSignal for 通知，Twilio for 短信）。
