# 事件追踪网站 - 系统架构方案

## 1. 系统架构概览

### 1.1 架构设计原则
- **微服务架构**: 前后端分离，独立部署和扩展
- **云原生设计**: 基于容器化部署，支持弹性伸缩
- **安全性优先**: 多层安全防护，数据加密传输和存储
- **高可用性**: 99.9%可用性保障，故障自动恢复
- **可扩展性**: 支持水平扩展和垂直扩展

### 1.2 技术栈选择

#### 前端技术栈
```
框架层: React 18 + TypeScript 5.x
状态管理: React Context + useReducer + Zustand
路由管理: React Router v6
UI框架: TailwindCSS + HeadlessUI
动画库: Framer Motion
图标库: React Icons
HTTP客户端: Axios + React Query
表单处理: React Hook Form + Zod
日期处理: Day.js
图表库: Chart.js + react-chartjs-2
```

#### 后端技术栈
```
运行时: Node.js 18 LTS + TypeScript
框架: Express.js + Socket.io
数据库: PostgreSQL 14 + Redis 7
对象存储: Supabase Storage
认证授权: JWT + Passport.js
API文档: Swagger/OpenAPI
任务调度: Bull Queue + Redis
邮件服务: Nodemailer
推送服务: Web Push + Server-Sent Events
```

#### 第三方服务集成
```
认证服务: 微信开放平台API
AI服务: 文心一言ERNIE Bot API
云存储: Supabase (PostgreSQL + Storage)
CDN: Vercel Edge Network
监控服务: Sentry + LogRocket
分析服务: Google Analytics
```

## 3. 数据库设计

### 3.1 数据模型设计

#### 用户表 (users)
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    wechat_openid VARCHAR(100) UNIQUE NOT NULL,
    wechat_unionid VARCHAR(100),
    nickname VARCHAR(100),
    avatar_url TEXT,
    email VARCHAR(255),
    phone VARCHAR(20),
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP
);

CREATE INDEX idx_users_wechat_openid ON users(wechat_openid);
CREATE INDEX idx_users_created_at ON users(created_at);
```

#### 事件表 (events)
```sql
CREATE TABLE events (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    type VARCHAR(50) NOT NULL CHECK (type IN ('normal', 'ai_push')),
    priority VARCHAR(20) CHECK (priority IN ('high', 'medium', 'low')),
    progress INTEGER CHECK (progress >= 0 AND progress <= 100),
    status VARCHAR(50) DEFAULT 'pending' CHECK (status IN ('pending', 'in_progress', 'completed', 'cancelled')),
    start_date DATE,
    end_date DATE,
    reminder_enabled BOOLEAN DEFAULT false,
    reminder_frequency VARCHAR(50) CHECK (reminder_frequency IN ('once', 'daily', 'weekly', 'monthly')),
    reminder_method VARCHAR(50) CHECK (reminder_method IN ('email', 'notification', 'sms')),
    images TEXT[], -- JSON array of image URLs
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_events_user_id ON events(user_id);
CREATE INDEX idx_events_type ON events(type);
CREATE INDEX idx_events_status ON events(status);
CREATE INDEX idx_events_start_date ON events(start_date);
```

#### AI推送配置表 (ai_push_configs)
```sql
CREATE TABLE ai_push_configs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    event_id UUID REFERENCES events(id) ON DELETE CASCADE,
    push_frequency VARCHAR(50) DEFAULT 'daily' CHECK (push_frequency IN ('realtime', 'hourly', 'daily', 'weekly')),
    push_method VARCHAR(50) DEFAULT 'notification' CHECK (push_method IN ('in_app', 'email', 'wechat')),
    info_type VARCHAR(50) DEFAULT 'comprehensive' CHECK (info_type IN ('news', 'tech', 'comprehensive')),
    push_quantity INTEGER DEFAULT 10 CHECK (push_quantity > 0),
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_ai_push_configs_user_id ON ai_push_configs(user_id);
CREATE INDEX idx_ai_push_configs_event_id ON ai_push_configs(event_id);
```

#### AI内容表 (ai_contents)
```sql
CREATE TABLE ai_contents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    event_id UUID REFERENCES events(id) ON DELETE CASCADE,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    source_url TEXT,
    relevance_score DECIMAL(3,2) CHECK (relevance_score >= 0 AND relevance_score <= 1),
    quality_score DECIMAL(3,2) CHECK (quality_score >= 0 AND quality_score <= 1),
    is_processed BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_ai_contents_event_id ON ai_contents(event_id);
CREATE INDEX idx_ai_contents_created_at ON ai_contents(created_at);
CREATE INDEX idx_ai_contents_relevance_score ON ai_contents(relevance_score);
```

## 5. 数据流设计

### 5.1 用户认证流程
```
1. 用户访问登录页面
2. 前端请求二维码: GET /api/auth/wechat/qr
3. 后端生成场景ID和二维码URL
4. 前端展示二维码并轮询状态: GET /api/auth/wechat/status/{sceneId}
5. 用户扫码确认登录
6. 后端验证并生成JWT令牌
7. 前端获取用户信息并存储
8. 跳转到事件展示页面
```

### 5.2 事件创建流程
```
1. 用户在创建页面填写事件信息
2. 前端表单验证 (React Hook Form + Zod)
3. 发送创建请求: POST /api/events
4. 后端数据验证和清洗
5. 存储到PostgreSQL
6. 清除相关缓存 (Redis)
7. 返回创建结果
8. 前端更新本地状态和UI
```

### 5.3 AI推送流程
```
1. 用户创建AI推送类型事件
2. 系统自动触发分析任务
3. 调用文心一言API进行内容分析
4. 爬取相关信息和数据
5. 内容质量评估和去重
6. 存储到数据库
7. 根据配置推送通知
8. 用户可在数据整合页面查看
```

## 6. 安全设计

### 6.1 认证安全
- **JWT令牌**: 使用RS256算法，有效期2小时
- **刷新令牌**: 有效期7天，存储在HttpOnly Cookie
- **密码策略**: 强密码要求，定期更换提醒
- **多因素认证**: 支持2FA (未来扩展)

### 6.2 数据安全
- **传输加密**: 全站HTTPS，TLS 1.3
- **存储加密**: 敏感数据AES-256加密
- **数据备份**: 每日自动备份，异地存储
- **访问控制**: 基于角色的权限管理 (RBAC)

### 6.3 API安全
- **速率限制**: 基于IP和用户的双重限制
- **输入验证**: 严格的参数验证和SQL注入防护
- **CORS策略**: 白名单机制
- **请求签名**: 重要操作需要签名验证

## 7. 性能优化策略

### 7.1 前端优化
- **代码分割**: 路由级别的懒加载
- **资源压缩**: Gzip/Brotli压缩，图片WebP格式
- **缓存策略**: 浏览器缓存 + Service Worker
- **CDN加速**: 静态资源全球分发

### 7.2 后端优化
- **数据库优化**: 索引优化，查询优化，连接池
- **缓存策略**: 多层缓存 (Redis + 应用缓存)
- **异步处理**: 耗时操作异步化 (Bull Queue)
- **数据库分片**: 按用户ID分片 (未来扩展)

### 7.3 监控和告警
- **应用监控**: Sentry错误监控，性能监控
- **基础设施监控**: CPU、内存、磁盘、网络
- **业务指标监控**: 用户活跃度，功能使用频率
- **告警机制**: 多渠道告警 (邮件、短信、钉钉)

## 8. 部署架构

### 8.1 容器化部署
```dockerfile
# 前端Dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

- **配置备份**: Git版本控制 + 配置中心
