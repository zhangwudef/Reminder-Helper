# 事件追踪网站 - 接口规范与数据流文档

## 1. 接口规范总览

### 1.1 设计原则
- **RESTful风格**: 符合REST架构风格，资源导向
- **统一格式**: 统一的请求和响应格式
- **版本控制**: API版本管理，向后兼容
- **安全性**: 认证授权，数据加密
- **高性能**: 缓存策略，分页机制

### 1.2 基础规范

#### 请求规范
```
Base URL: https://api.event-tracker.com/api/v1
Content-Type: application/json
Accept: application/json
Authorization: Bearer {access_token}
```

#### 响应规范
```json
{
  "success": true,
  "code": 200,
  "message": "操作成功",
  "data": {},
  "pagination": {
    "page": 1,
    "limit": 20,

    "hasNext": true
  },
  "timestamp": "2024-01-01T00:00:00.000Z",
  "requestId": "uuid-v4",
  "version": "1.0.0"
}
```

#### 错误响应规范

```json
{
```

## 2. 认证授权接口

### 2.1 微信登录接口

#### 获取登录二维码
```http
GET /auth/wechat/qr
```

**请求参数**:
```json
{
  "deviceId": "string", // 设备ID，用于区分不同设备
  "redirectUri": "string" // 可选，登录成功后的重定向地址
}
```

**响应数据**:
```json
{
  "success": true,
  "code": 200,
  "data": {
```

## 8. WebSocket接口

### 8.1 实时通信

#### WebSocket连接
```javascript
const ws = new WebSocket('wss://api.event-tracker.com/ws');

// 认证
ws.send(JSON.stringify({
  type: 'auth',
  token: 'Bearer {access_token}'
}));

// 订阅事件更新
ws.send(JSON.stringify({
  type: 'subscribe',
  channel: 'events'
}));

// 接收消息
ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  switch(data.type) {
    case 'event.created':
      console.log('新事件创建:', data.payload);
      break;
    case 'event.updated':
      console.log('事件更新:', data.payload);
      break;
    case 'ai.content.ready':
      console.log('AI内容就绪:', data.payload);
      break;
  }
};
```

#### 消息格式
```json
{
  "type": "event.created",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "requestId": "uuid-v4",
  "payload": {
    "eventId": "uuid-v4",
    "title": "新事件标题",
    "type": "normal"
  }
}
```

## 10. 数据流设计

### 10.1 用户认证数据流
```
用户访问 → 前端路由守卫 → 检查认证状态 → 未认证 → 跳转登录页
                ↓                           ↓
            已认证 ← 存储用户信息 ← 验证成功 ← 调用登录API
                ↓
            访问受保护资源
```

### 10.2 事件管理数据流
```
用户操作 → 前端表单验证 → 发送API请求 → 后端验证 → 数据库操作
                ↓              ↓           ↓         ↓
            显示结果 ← 更新本地状态 ← 返回响应 ← 缓存更新
```

### 10.3 AI推送数据流
```
创建AI事件 → 触发分析任务 → 调用文心一言API → 内容处理 → 存储结果
                ↓              ↓           ↓         ↓
            推送通知 ← 调度系统 ← 质量评估 ← 内容爬取
```

### 10.4 实时通信数据流
```
用户操作 → WebSocket发送 → 服务器处理 → 广播消息 → 客户端接收
                ↓              ↓           ↓         ↓
            状态更新 ← 订阅频道 ← 权限验证 ← 连接认证
```

## 11. 缓存策略

### 11.1 接口缓存规则

#### 客户端缓存
```
GET /events → Cache-Control: private, max-age=300 // 5分钟
GET /calendar/* → Cache-Control: private, max-age=900 // 15分钟
GET /statistics/* → Cache-Control: private, max-age=1800 // 30分钟
GET /auth/profile → Cache-Control: private, no-cache // 不缓存
```

#### 服务器端缓存 (Redis)
```
events:list:{userId}:{hash} → TTL: 300s
event:detail:{eventId} → TTL: 600s
calendar:{userId}:{year}:{month} → TTL: 900s
statistics:{userId}:{type} → TTL: 1800s
user:profile:{userId} → TTL: 3600s
```

### 11.2 缓存失效策略

#### 主动失效
- 事件创建/更新/删除时清除相关缓存
- 用户资料更新时清除用户缓存
- AI内容更新时清除统计缓存

#### 被动失效
- TTL过期自动失效
- LRU算法自动清理
- 内存不足时优先清理
